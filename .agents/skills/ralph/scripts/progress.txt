## Codebase Patterns
- Next.js config is at `next.config.ts` (TypeScript, uses `NextConfig` type)
- tRPC server context is in `src/trpc/server.ts`, uses `cache()` wrapper from React
- Environment validation is in `src/env.js` and imported in `next.config.ts`
- Project uses `~` path alias for `src/` directory
- This is a T3 stack app (Next.js + tRPC + Prisma + NextAuth)
- Prisma 7 generated client lives at `prisma/generated/client` ‚Äî import PrismaClient from there, not `@prisma/client` (do NOT use `.js` extension ‚Äî Turbopack can't resolve it)
- Tailwind CSS 4 uses CSS-first config: `@import "tailwindcss"` in globals.css, `@theme inline {}` for custom theme values, `@custom-variant dark` for dark mode
- PostCSS config uses `@tailwindcss/postcss` plugin (not `tailwindcss`)
- `tailwindcss-animate` replaced with `tw-animate-css` (v4 compatible) ‚Äî import via `@import "tw-animate-css"` in globals.css
- Next.js 16 config: use `import "./src/env.js"` (not `await import()`) ‚Äî top-level await breaks Turbopack's require()
- Prisma 7 requires a driver adapter (e.g., `PrismaPg` from `@prisma/adapter-pg`) passed to PrismaClient constructor
- `prisma.config.ts` at repo root handles datasource URL and migration config (replaces `url` in schema.prisma)
- Use `@prisma/adapter-pg` for regular PostgreSQL, NOT `@prisma/adapter-ppg` (that's for Prisma Postgres managed service)
- Auth is next-auth v5 (Auth.js): `NextAuth()` returns `{ handlers, auth, signIn, signOut }` ‚Äî use `auth()` instead of `getServerSession()`
- Discord provider in v5: `import Discord from "next-auth/providers/discord"` (not `DiscordProvider`)
- Auth route handler: `export const { GET, POST } = handlers` from `~/server/auth`
- Auth.js v5 env vars: use `AUTH_SECRET` and `AUTH_URL` (not `NEXTAUTH_*`)
- tRPC v11 stable: use `httpBatchStreamLink` (not `unstable_httpBatchStreamLink`)
- ESLint 9 flat config in `eslint.config.mjs` ‚Äî uses `typescript-eslint` unified package (not separate `@typescript-eslint/parser` + `@typescript-eslint/eslint-plugin`)
- `next lint` removed in Next.js 16 ‚Äî use `eslint .` directly in lint script
- `eslint-config-next` v16 exports flat config arrays (can be spread into `tseslint.config()`)
